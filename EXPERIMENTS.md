# Experiments

This file consolidates the experiment record that was previously spread across notebooks, scripts, and individual class-project folders.

## Task

The project uses the Kaggle competition [Classification of Math Problems by Kasut Academy](https://www.kaggle.com/competitions/classification-of-math-problems-by-kasut-academy/overview). Each input is a math problem, and the model predicts one of eight labels:

| Label | Category |
| --- | --- |
| 0 | Algebra |
| 1 | Geometry and Trigonometry |
| 2 | Calculus and Analysis |
| 3 | Probability and Statistics |
| 4 | Number Theory |
| 5 | Combinatorics and Discrete Math |
| 6 | Linear Algebra |
| 7 | Abstract Algebra and Topology |

## Data And Augmentation

- Original data comes from the Kaggle competition and is expected at `data/train.csv` and `data/test.csv`.
- `data/train_augmented.csv`, `data/train_pp.csv`, and `data/test_pp.csv` are derived data files used by some notebooks.
- [`notebooks/01_data_generation/data_generation_generic.ipynb`](notebooks/01_data_generation/data_generation_generic.ipynb) and [`notebooks/01_data_generation/data_generation_specific.ipynb`](notebooks/01_data_generation/data_generation_specific.ipynb) document AWS Bedrock data generation.
- [`notebooks/04_llm_experiments/nlp-final-project-nn-llms.ipynb`](notebooks/04_llm_experiments/nlp-final-project-nn-llms.ipynb) documents a Gemma/vLLM paraphrase-generation path. This is best understood as LLM-assisted data generation, not the primary classifier comparison.
- [`src/llmvsslm/augment.py`](src/llmvsslm/augment.py) contains a local TextAttack-based augmentation path.

## Caveats

- Public scores are documented in notebooks/reports and were not recomputed during repository cleanup.
- Several notebooks were originally run in Kaggle, AWS, or local GPU environments and may include path assumptions in markdown or cells.
- The Llama notebook includes an internal holdout check that appears to print a count of correct predictions rather than an accuracy rate. Normalize this value before comparing it with other metrics.
- Bedrock and vLLM/Gemma data generation are not fully deterministic unless prompts, generated outputs, seeds, model versions, and provider settings are archived.
- For a strict paper-style reproduction, record package versions, CUDA version, GPU model, random seeds, and artifact checksums for each run.
